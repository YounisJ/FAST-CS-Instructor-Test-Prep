# Mock Tests – Full Exam Simulation

This folder contains **full-length MCQ mock tests** designed to simulate the actual FAST screening exam.

---

## Purpose

Mock tests are used to:

- Simulate real exam pressure
- Improve time management
- Test cross-subject knowledge
- Identify weak areas under exam conditions

---

## Test Format

Each mock test contains:

- 100 MCQs
- Mixed subjects
- Medium to advanced difficulty
- No negative marking simulation
- Answer keys with explanations

---

## Subjects Mixed in Each Test

- Programming Fundamentals
- OOP
- DSA
- DBMS
- OS
- Discrete Structures
- AI
- Automata
- Software Engineering
- Computer Networks

---

## Time Strategy

Recommended time:

- 90–120 minutes per test

---

## How to Attempt Mock Tests

### STEP 1: Full Simulation
- Sit like real exam
- No interruption
- No notes

### STEP 2: Evaluation
- Check answer key immediately
- Mark wrong answers

### STEP 3: Error Analysis
- Identify weak topics
- Go back to `/notes`

### STEP 4: Revision Loop
- Re-attempt wrong questions after 2–3 days

---

## Progress Strategy

- Mock Test 1–2 → Learning phase
- Mock Test 3–4 → Performance improvement
- Mock Test 5 → Final exam readiness

---

## Rule

Do NOT attempt mock tests before completing:

- Basic notes revision
- At least 5–8 mini tests per subject
